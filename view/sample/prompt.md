# 論理的議論支援アプリケーションの UI 設計プロンプト

## 概要

GitHub のイシュー・プルリクエストの議論スタイルを参考に、構造化された論理的議論を支援する Web アプリケーションの UI を設計してください。

## デザインテーマ

- GitHub の UI デザイン言語をベースにする
- クリーンでモダンな外観
- 階層構造を視覚的に理解しやすいデザイン

## 主要画面

### 1. 議論一覧画面

- GitHub のイシュー一覧のようなレイアウト
- フィルタリング機能(公開レベル、作成者、日付など)
- 各議論カードに以下を表示:
  - タイトル(テーマ)
  - コメント数
  - 作成者のアバター
  - 作成日時
  - ステータスラベル(公開レベル、コメント許可レベル)

### 2. 議論詳細画面

- GitHub のイシュー詳細画面に類似
- 上部にテーマと説明を表示
- コメントを木構造で表示(インデントで親子関係を表現)
- 子コメントは分かりやすくコメントの種類ごとに分けられている
- 各コメントカードには:
  - 投稿日時
  - 内容
  - 返信ボタン
  - 状態表示(割り当て待ち/割り当て済み/アーカイブ)

### 3. コメント作成/編集フォーム

- GitHub のコメント入力エリアに類似
- コメント種類の選択ドロップダウン
- Markdown エディタ(プレビュー機能付き)
- ルールに基づいた選択可能なコメント種類のみ表示

### 4. ノート機能

- サイドパネルまたはモーダルで表示
- 簡易的なメモ入力欄
- ノート一覧(ドラッグ&ドロップでコメントに変換可能)

### 5. ルール設定画面

- コメント種類の定義(名前、説明、色)
- フローチャート風の経路図
- ノードとエッジで視覚化

### 6. 指摘(Issue)表示

- コメントに対する警告表示
- GitHub のレビューコメントのような吹き出し
- 矛盾や循環論法を視覚的に強調

## UI 要素

### カラースキーム

- プライマリ: GitHub 風の青系統
- セカンダリ: グレースケール
- アクセント: 警告=オレンジ、エラー=赤、成功=緑
- コメント種類ごとに異なる色のバッジ

### アイコン

- コメント: チャットバブル
- ノート: 付箋
- 指摘: 警告マーク
- ルール: フローチャート
- グループ: 複数人のアイコン

### インタラクション

- ホバー時のツールチップ
- コメント種類選択時にルールに基づいたヘルプテキスト
- ドラッグ&ドロップでノートをコメントに変換
- 木構造の展開/折りたたみ

## レスポンシブ対応

- デスクトップ: 3 カラムレイアウト(サイドバー + メイン + 情報パネル)
- タブレット: 2 カラムレイアウト
- モバイル: 1 カラムレイアウト、ハンバーガーメニュー

上記の仕様に基づいて、モダンでユーザーフレンドリーな UI を設計してください。
